\section{Challenges we faced}
\begin{enumerate}
    \item Finding proper dataset for our usecase: Initially we used sisfall dataset but the sensor used to create this dataset is not same as our sensor.
    So we manually collect the data and labelled the data with our sensor.

    \item Selection of ml model:Since our dataset was prepared manually due to constrain of resources we are only able to collect 1000 datapoints, so due to lack of knowledge about different models performance on small dataset we were not able to prioritise which model to use initially.
    So we trained regression model, decision tree, random forest, Xg boost, neural network  models and decided based on the accuracy of model.

    \item Preparing our own dataset: We need to tag fall or nofall manually for every datapoint and initial most of the times when experiment is conducted if we connected module with battery , battery used to get disconnected so we connected usb to the esp32 and collected the data.
    \item Wifi connectivity and Power issue: While the system worked well when powered via a USB adapter, achieving stable operation when disconnected and running entirely on a battery unit proved difficult
    \item Frequent refreshing of page issue for frontend dashboard
\end{enumerate}
